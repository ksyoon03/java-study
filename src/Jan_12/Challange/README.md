# 문제 풀기

### ⚔️ 도전 과제: [전설의 아이템 상점 시스템]

형님은 게임 개발자가 되어, **다양한 종류의 아이템을 하나의 가방(배열)에서 관리하고 사용하는 시스템**을 구현해야 합니다.

### 1. 공통 설정 클래스 (`GameSetting`)

- **조건:** 게임 내에서 변하지 않는 상수를 정의하세요.
- **필드:**
    - `SHOP_NAME`: "전설의 대장간" (문자열, 상수)
    - `INVENTORY_SIZE`: 3 (정수, 상수)

### 2. 부모 클래스 (`Item`)

- 모든 아이템의 기본 설계도입니다.
- **필드:**
    - `name` (아이템 이름, 문자열, `protected`)
    - `price` (가격, 정수, `protected`)
- **생성자:** 이름과 가격을 입력받아 초기화하세요.
- **메서드:**
    - `use()`: "아이템을 사용했습니다."를 출력하는 기본 메서드.

### 3. 자식 클래스 1 (`Weapon`)

- `Item`을 상속받으세요.
- **필드:** `power` (공격력, 정수, `private`)
- **생성자:** 이름, 가격, 공격력을 입력받아 초기화 (부모 생성자 `super` 활용 필수).
- **메서드 오버라이딩:**
    - `use()`: "⚔️ [이름]을(를) 장착했습니다! (공격력 +공격력)" 출력.

### 4. 자식 클래스 2 (`Potion`)

- `Item`을 상속받으세요.
- **필드:** `healAmount` (회복량, 정수, `private`)
- **생성자:** 이름, 가격, 회복량을 입력받아 초기화.
- **메서드 오버라이딩:**
    - `use()`: "🍷 [이름]을(를) 마셨습니다." 출력.
- **고유 메서드:**
    - `heal()`: " => 체력이 [회복량]만큼 회복됩니다!" 출력.

### 5. 실행 클래스 (`ShopMain`)

- `main` 메서드에서 다음 로직을 작성하세요.
    1. `GameSetting`의 상수를 사용하여 "=== [상점 이름] ==="을 출력하세요.
    2. `Item` 타입의 배열을 `INVENTORY_SIZE`만큼 생성하세요. (**업캐스팅**)
    3. 배열의 각 칸에 다음 아이템들을 저장하세요.
        - 0번: `Weapon` ("무한의 대검", 3000원, 공격력 70)
        - 1번: `Potion` ("빨간 물약", 50원, 회복량 100)
        - 2번: `Weapon` ("낡은 도끼", 100원, 공격력 5)
    4. *반복문(for문)**을 사용하여 배열에 있는 모든 아이템의 `use()` 메서드를 호출하세요.
    5. **조건문과 캐스팅**을 활용하여:
        - 현재 꺼낸 아이템이 **물약(`Potion`)**이라면, 물약만의 고유 기능인 `heal()` 메서드를 추가로 호출하세요. (**다운캐스팅 & instanceof**)

---

### 💻 출력 예시 (목표 결과)

```jsx
=== 전설의 대장간 ===
⚔️ [무한의 대검]을(를) 장착했습니다! (공격력 +70)
-------------------------
🍷 [빨간 물약]을(를) 마셨습니다.
   => 체력이 100만큼 회복됩니다!
-------------------------
⚔️ [낡은 도끼]을(를) 장착했습니다! (공격력 +5)
-------------------------

```
